# Release Notes - Version 1.0.0

## Released: 2026-06-21

### 🎉 What's New

- **Interactive TUI build menu** — running `./scripts/bash/00.build_crkbd_fw.sh` with no arguments
  now shows a full-screen arrow-key navigator. Use Up/Down to move the selection, Enter to confirm,
  or `q` to quit. "Both sides" is pre-selected as the default — the most common build target.
- **Live build output** — a rolling 12-line log tail with a braille spinner and elapsed-time counter
  is displayed during each build, updating in-place without scrollback noise. The display clears
  automatically on completion.
- **Comprehensive Quick Start guide** — `README.md` now contains a step-by-step setup walkthrough
  covering Docker Desktop prerequisites, WSL2 integration, Dev Container setup, West workspace
  initialisation, and manual `west build` commands with a full flag reference.

### ✨ Improvements

- **Board target corrected to `nice_nano//zmk`** — all build commands (script and manual examples in
  `README.md`) now use the ZMK-specific board variant. The old `nice_nano` / `nice_nano_v2` targets
  do not apply RGB underglow and ext-power overlays, causing `undefined node label` errors at
  configure time.
- **`scripts/README.md` rewritten** — focused on day-to-day build script usage with a clean argument
  table, output file listing, feature list, and prerequisites section.
- **VS Code workspace configured** — workspace settings updated with Prettier formatter and
  markdownlint extension recommendations for consistent code and documentation formatting.

### 🔧 Bug Fixes

- **Banner box misalignment fixed** — the TUI header box (`╔═══╗`) now renders as a proper rectangle
  on Docker/Linux terminals. The previous em dash in the title string (`—`) was counted as 3 bytes
  in a C-locale shell, inflating the padding calculation and making the bottom border shorter than
  the top.
- **Duplicate "Building..." header during live log** — when log line count decreased between refresh
  ticks, the cursor-up offset was under-counted, causing the spinner header to be printed mid-screen
  and appear twice simultaneously. The `prev` counter now only ever increases, maintaining correct
  cursor distance from the header at all times.
- **Ctrl+C now cleans up correctly** — an interrupt handler kills the background `west build`
  process, clears the live display area, and prints a short "Interrupted." message before exiting.
  Previously the spinner could be left on screen and the background process would continue running.
- **Outdated setup docs removed** — `00.BUILD_NOTE.md` and `00.SETUP.md` contained stale
  instructions that contradicted the current workflow; both files have been removed and their
  content replaced by the expanded Quick Start section in `README.md`.
